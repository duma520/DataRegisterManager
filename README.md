## **1. 前言：为什么需要这个系统？**

在日常工作、学习、科研、管理中，我们经常需要记录各种**结构化数据**：

- 学生登记表
- 设备维护日志
- 会议签到记录
- 实验数据表格
- 客户信息台账
- 个人日记/笔记（按日期分类）

传统方式如 Excel 虽然灵活，但存在以下问题：
- 多人协作困难
- 数据易丢失
- 无自动备份
- 无法按用户隔离
- 缺乏登录审计

**资料登记管理系统** 正是为解决这些问题而生。它是一个**轻量级、跨平台、单文件部署**的桌面应用，基于 Python + SQLite + PySide6 构建，无需安装数据库服务器，开箱即用。

## **3. 快速入门指南（零基础版）**

### **3.1 安装与启动**
1. 下载 `DataRegisterManager.exe`（Windows）或 `.py` 文件（需 Python 环境）
2. 双击运行（Windows）或执行 `python DataRegisterManager.py`
3. 首次启动会自动创建数据库和备份目录

### **3.2 首次登录**
- 系统默认创建一个管理员账户：
  - **用户名**：`admin`
  - **密码**：`admin123`
- 在登录界面输入后点击“登录”

### **3.3 基本操作**
1. **选择单元格**：顶部下拉框选择“单元格 1”、“单元格 2”等
2. **输入数据**：直接在表格中点击单元格输入文字
3. **自动保存**：输入后立即保存，无需手动点击“保存”
4. **切换用户**：菜单栏 → 用户 → 切换用户

> 💡 **小技巧**：  
> - 按 `Ctrl+F` 或点击工具栏“搜索”可全文检索  
> - 关闭窗口时数据已自动保存，无需担心丢失

## **4. 详细功能说明（中级用户）**

### **4.1 用户管理**
- **添加用户**：登录界面 → “用户管理” → “添加用户”
  - 可设置密码（留空则无需密码）
- **编辑用户**：修改用户名或密码
- **删除用户**：**注意！此操作不可逆，会删除该用户所有数据**

### **4.2 页面配置**
在“用户配置”中可设置：
- **页面行数/列数**：决定每页显示多少个单元格（如 2×3 = 6 个单元格）
- **自动调整列宽**：勾选后列宽会根据内容自动伸缩

### **4.3 单元格配置**
每个单元格可独立配置（通过右键菜单或“编辑配置”）：
| 配置项 | 说明 |
|--------|------|
| 行数/列数 | 表格大小（最大支持 1000×1000） |
| 需要标题 | 是否显示表头 |
| 标题模式 | 统一标题（所有列相同）或独立标题（每列可不同） |
| 需要登录时间 | 记录每次输入的时间戳 |
| 计算相隔天数 | 显示距离登录时间的天数（适用于签到类场景） |

> 📌 **示例**：  
> 若用于“每日体温登记”，可设置：  
> - 行数=31（天数），列数=1  
> - 标题=“体温(℃)”  
> - 启用“登录时间”和“相隔天数”  

### **4.4 搜索功能**
- 支持中文、英文、拼音首字母搜索
- 搜索结果会高亮显示匹配内容的位置（单元格+行列坐标）

## **5. 高级配置与自定义（专业用户）**

### **5.1 自定义列标题**
1. 右键点击表格 → “设置列标题”
2. 为每一列输入独立标题（如“姓名”、“电话”、“状态”）
3. 支持“统一设置”一键填充相同标题

### **5.2 调试模式**
- 菜单栏 → 视图 → 勾选“调试模式”
- 状态栏会显示详细操作日志（如“数据已保存 - 单元格1 (2,3)”）

### **5.3 配置文件位置**
- 所有数据存储在程序同目录的 `资料登记管理系统.db`
- 备份文件位于 `backups/` 文件夹
- 配置缓存：`QSettings`（Windows 注册表 / Linux ~/.config）

## **6. 数据库与备份机制详解**

### **6.1 数据库特性**
- **SQLite WAL 模式**：支持高并发写入，崩溃后自动恢复
- **表结构**：
  - `users`：用户信息（含密码哈希）
  - `data_records`：核心数据表（含用户ID、单元格索引、行列坐标、内容、时间戳）
  - `cell_templates`：单元格配置模板

### **6.2 备份策略**
| 备份类型 | 触发条件 | 保留数量 |
|----------|----------|----------|
| 手动备份 | 用户点击“手动备份” | 最多 30 个 |
| 自动备份 | 每小时一次 | 最多 30 个 |
| 回滚备份 | 恢复备份前自动创建 | 临时保留 |

> 🔒 **恢复备份流程**：  
> 1. 文件 → 备份管理  
> 2. 选择备份文件 → 点击“恢复”  
> 3. 系统会先创建回滚点，再覆盖当前数据库

## **7. 多用户与权限管理**

- **用户隔离**：每个用户只能看到自己的数据
- **密码安全**：
  - 非空密码使用 SHA-256 哈希存储
  - 空密码允许快速登录（适合单人使用场景）
- **默认管理员**：首次运行自动创建 `admin/admin123`，建议首次登录后修改密码

## **8. 性能优化与大规模数据处理建议**

### **8.1 性能瓶颈**
- 单表格超过 1000 行 × 1000 列可能导致界面卡顿
- 搜索功能对超大数据集（>10万条）响应较慢

### **8.2 优化建议**
- **分单元格存储**：将大表拆分为多个单元格（如按月份分）
- **关闭自动列宽**：在“用户配置”中取消勾选“自动调整列宽”
- **定期清理**：通过 SQLite 工具手动删除旧数据

## **9. 故障排查与常见问题**

| 问题 | 解决方案 |
|------|----------|
| 启动报错“缺少 DLL” | 安装 [Microsoft Visual C++ Redistributable](https://aka.ms/vs/17/release/vc_redist.x64.exe) |
| 数据未保存 | 检查磁盘空间是否充足，或尝试重启应用 |
| 备份恢复失败 | 确保目标备份文件未被其他程序占用 |
| 中文显示乱码 | 确保系统区域设置为中文（UTF-8） |

> ⚠️ **重要**：  
> 如遇数据库损坏，请立即从 `backups/` 目录恢复最新备份！

## **11. 开发者指南（二次开发）**

- Python 3.8+
- PySide6
- SQLite3
